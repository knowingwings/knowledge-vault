# Side projects: psychological release valve vs. focus dilution

## Premise
Surfaced 03/05/2026 while reflecting on a Sunday Minecraft playthrough (Create+ modpack) reminding me of an old YouTuber (ahchx1, ~14 years ago). The instinct was: "this style of video would take off now — I should start a separate channel just for fun."

The pattern is recognisable: starting a new low-stakes creative project at the moment the high-stakes one (knowingslab Reducer video) is grinding through a slip cycle.

## The question
**When does a side project help and when does it hurt?**

Help case (release valve):
- Lower-stakes mode lets the brain stop white-knuckling the high-stakes thing.
- "Just for fun" content with no expectation removes the public-facing resistance that's blocking the main channel.
- Different audience / different mode might surface skills that transfer back.
- Sustainable creative output requires variety; one pillar project burns out solo.

Hurt case (focus dilution):
- Knowingslab has 0 videos shipped. A second channel before the first lands is the textbook anti-pattern Q2 explicitly listed under "NOT doing."
- "Low effort" is the same phrase that hid Reducer's real scope (12 sub-ticks under one tick).
- Cognitive switching cost is high; finishing one thing properly often beats two things half-done.
- Resistance to the main thing might just be transferred to the new thing once it has stakes.

## Useful diagnostic questions
- Does the new project require *infrastructure* (channel setup, branding, posting cadence) or only *content* (recording sessions on existing infrastructure)? Infrastructure is the warning sign.
- Could the same idea sit *inside* the existing project as a sub-series? Often "I want to do X separately" is a way of avoiding the question "does this fit in what I'm already doing?"
- Is the new thing genuinely lower-stakes, or just lower-stakes-feeling because the resistance hasn't kicked in yet?

## Connections
- `user_youtube_slip_pattern.md` — first-video resistance + scope hiding.
- Q2 sprint NOT-doing list — explicit statement that new project kickoffs before finishing live ones is a known failure mode.
- The momentum insight (28/04, 29/04) — easier to keep going on what's started than cold-start something new. A second channel is a cold-start.

## Status
Seed. Don't decide on the second channel until Reducer + Algernon are shipped (i.e. the main channel has a real existence). At that point, the question can be asked honestly.
